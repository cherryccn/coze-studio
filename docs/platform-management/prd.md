# 平台管理产品需求文档（PRD）

## 1. 背景与目标
当前系统已有空间管理、发布管理、模型管理能力，但缺少平台视角的成本与运营数据统一入口。  
本模块目标是将「计费管理 + 统计分析」放入左侧菜单一级入口，形成平台级治理闭环。

## 2. 产品目标
1. 平台管理员可快速查看成本、用量、趋势与异常。
2. 支持按空间、项目类型、时间范围做统计和账单查询。
3. 支持预算阈值设置与超限告警。
4. 与现有发布管理、空间管理的数据模型打通，避免重复建设。

## 3. 非目标（MVP 不做）
1. 不做在线支付、充值、发票流程。
2. 不做多币种结算。
3. 不做复杂的分账和二级代理结算。

## 4. 入口与权限
## 4.1 菜单入口
- 左侧菜单新增：`平台管理`
- 位置建议：`空间配置` 下方、`系统管理` 相关入口同级（按你们现有菜单层级适配）

## 4.2 可见权限
| 角色 | 是否可见平台管理 | 说明 |
|---|---|---|
| platform_admin | 是 | 全量可见、可配置 |
| org_owner/org_admin | 是（可选） | 若后续引入组织维度，可按组织范围隔离 |
| space_owner/admin/member | 否 | 仍通过空间管理查看空间级配置 |

## 5. 页面信息架构
一级页：`平台管理`  
二级 Tab：
1. `计费管理`
2. `统计模块`

共用筛选区：
- 时间范围：今天/近7天/近30天/自定义
- 空间：全部空间 + 指定空间
- 项目类型：智能体/应用/工作流/全部

## 6. 计费管理需求
## 6.1 费用总览
展示指标卡：
- 今日费用
- 本月累计费用
- Token 消耗
- 活跃空间数

展示图表：
- 费用趋势（按天）
- Token 趋势（按天）
- Top 空间成本排行

## 6.2 账单明细
明细字段：
- 时间
- 空间名称
- 项目类型
- 项目名称
- 模型/能力项
- 用量
- 单价
- 金额
- 状态（成功/失败/回滚）

能力要求：
- 支持搜索（空间名/项目名）
- 支持排序（时间、金额、用量）
- 支持导出 CSV

## 6.3 预算与阈值
配置项：
- 空间月预算上限
- 告警阈值（70%/90%/100%）
- 超限策略（仅告警/拒绝新增调用）

告警渠道（MVP）：
- 站内消息

## 7. 统计模块需求
## 7.1 使用概览
- 活跃空间数（DAU/WAU）
- 项目活跃数（智能体/应用/工作流）
- 调用总量
- 成功率
- 平均时延
- 总 Token 消耗

## 7.2 发布与效果分析
- 最近发布时间分布
- 发布后调用趋势
- 项目排行（调用量、Token、费用、失败率）

## 7.3 模型与能力分析
- 模型消耗排行（Token/费用）
- 能力项消耗排行（例如插件调用、工作流执行）
- 异常波动识别（相对近7天均值）

## 8. 关键口径定义
1. `token_consumption = prompt_tokens + completion_tokens`
2. `cost_amount = token_consumption * unit_price + extra_charge`
3. `active_space`：统计周期内有至少一次成功调用的空间
4. `success_rate = success_count / total_count`
5. `avg_latency_ms = total_latency_ms / success_count`

## 9. 交互与状态
1. 空状态：提示“暂无数据”，给出筛选重置按钮。
2. 错误状态：展示错误文案 + 重试按钮。
3. 加载状态：骨架屏或 loading 条。
4. 导出状态：导出任务异步化，提示导出中和完成态。

## 10. 埋点与观测
关键埋点：
- 页面曝光
- 筛选条件变更
- 导出点击
- 预算规则保存
- 告警触发

建议字段：
- operator_id
- selected_time_range
- selected_space_count
- tab_key

## 11. 验收标准（MVP）
1. 平台管理员可进入入口并看到两个 Tab。
2. 可按时间/空间/项目类型筛选并正确刷新数据。
3. 账单明细支持排序、搜索、导出。
4. 预算阈值可保存并能触发站内告警。
5. 关键口径与后端返回一致，前后端验收口径统一。
